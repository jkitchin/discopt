* Problem
:PROPERTIES:
:SOURCE: Markowitz (1952); Cornuejols & Tutuncu, "Optimization Methods in Finance"
:CLASS: QP
:KNOWN_OPTIMAL: 0.038 (scipy SLSQP reference)
:N_VARIABLES: 3
:N_CONSTRAINTS: 2
:END:

\begin{align}
\min_{w} \quad & w^T S w \\
\text{s.t.} \quad & \sum_i w_i = 1 \\
& \mu^T w \geq 0.10 \\
& 0 \leq w_i \leq 1
\end{align}

where $S$ is the covariance matrix and $\mu = [0.08, 0.10, 0.12]$.

All three backends report obj=0.0321, but the actual variance at the
solution point is 0.0381 (confirmed by direct numpy computation). The
solution POINT is correct (w=[0.321, 0.357, 0.321], return=0.10, sum=1.0),
but the reported objective value is wrong.

* Performance Comparison

All backends report identical results (QP fast path). No regression.

* Cross-Validation

scipy.optimize.minimize (SLSQP) finds obj=0.038 at w=[1/3, 1/3, 1/3].
Direct computation w @ S @ w at discopt's solution gives 0.0381, confirming
the objective value 0.0321 is incorrect.

The discopt-reported value (0.0321) matches the diagonal-only quadratic
form: S[0,0]*w1^2 + S[1,1]*w2^2 + S[2,2]*w3^2 = 0.0321. This means the
QP problem classifier is extracting only diagonal Hessian entries and
dropping the cross-terms (w1*w2, w1*w3, w2*w3).

* Verdict

SOLVER_BUG. The QP problem classifier drops off-diagonal quadratic terms
when extracting the Hessian from the expression DAG. The reported objective
is the diagonal-only computation, not the full quadratic form. The solution
point happens to be correct (the KKT conditions are similar), but the
objective value is wrong. This affects all QP problems with cross-terms.
