* Convexity Detection
:PROPERTIES:
:ARTICLE_TYPE: concept
:SOURCE_KEYS: adjiman1998 grant2006 khajavirad2012
:STATUS: draft
:CREATED: 2026-04-04
:END:

Convexity detection determines whether an optimization problem (or individual
functions within it) is convex, enabling the solver to select appropriate
algorithms. Convex problems can be solved to global optimality by local methods
(interior point, NLP-BB), while nonconvex problems require spatial branch and
bound with convex relaxations.

** Disciplined Convex Programming (DCP)

DCP applies composition rules to the expression DAG to verify convexity. Each
elementary function is classified as convex, concave, affine, or unknown, and
composition rules determine the convexity of composite expressions. For example,
exp(affine) is convex, sum of convex functions is convex, and the maximum of
convex functions is convex. DCP is conservative: it may classify a convex
function as "unknown" if the composition rules cannot verify convexity, but it
never misclassifies a nonconvex function as convex.

** Eigenvalue-Based Detection

For twice-differentiable functions, convexity can be verified by checking that
the Hessian is positive semidefinite everywhere on the domain. This is the
approach used in the alphaBB framework citep:adjiman1998, where the minimum eigenvalue
of the Hessian determines how much diagonal perturbation is needed to ensure
convexity. If the minimum eigenvalue is non-negative, the function is already
convex.

** Implementation in discopt

discopt implements DCP-style convexity detection on the expression DAG,
recognizing patterns like exp(affine), sum_of_convex, x**2, and others. When the
model is detected as convex, the solver auto-selects NLP-BB (which requires
convexity for valid lower bounds). When convexity cannot be verified, spatial B&B
with convex relaxations is used instead.

** Generalized Convexity (G-convexity)

DCP and eigenvalue tests certify ordinary convexity. A strictly larger class,
*convex-transformable* (G-convex) functions, becomes convex after a monotone
univariate transformation $G$ and still has the property that local minima are
global. citet:khajavirad2012 show that recognizing G-convexity of intermediate
sub-expressions yields convex relaxations provably tighter than the standard
factorable scheme — large root-gap and node-count reductions in branch-and-bound,
and the relaxation contributes a valid supporting-hyperplane cut usable directly
in an LP-based node relaxation. This is a concrete avenue for discopt to tighten
per-node bounds beyond DCP-convex recognition without ever risking an invalid
dual bound. See [[file:convex-transformable-functions.org][Convex-Transformable (G-convex) Functions]].
