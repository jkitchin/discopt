# C-38 fix — default-path false-optimal on `kall_circles_c8a` (2026-07-07)

**Verdict:** the false-optimal certificate is a **class-(a) unsound per-node
relaxation**: the in-house warm-started dual simplex returns a *numerical false*
`infeasible` on the ill-conditioned lifted McCormick relaxation of circle-packing
non-overlap constraints, **with no verified Farkas ray**, and the per-node LP path
**trusted that uncertified verdict** to fathom the node — removing the sub-box that
contains the true optimum and certifying a false-optimal dual bound
`3.6142 > 2.5409`. Fix: **an `infeasible` verdict may fathom a node only when backed
by a verified Farkas certificate; an uncertified `infeasible` is inconclusive (no
fathom)** — the sole rigorous proof of LP emptiness is a Farkas ray, not a
conditioning heuristic.

Branch `fix-c38-kall-circles` from `origin/main` (`8322bc87`). Python-only change
(`python/discopt/_jax/mccormick_lp.py`); no Rust touched. Run env
`PYTHONPATH=<worktree>/python JAX_PLATFORMS=cpu JAX_ENABLE_X64=1`.

## Reproduce (confirmed on this build, all flags OFF/default)

```
from_nl('kall_circles_c8a.nl').solve(time_limit=25, gap_tolerance=1e-4)
# main:  status=optimal  obj=3.6142347590  bound=3.6142347420  nodes=3   <- FALSE
```
MINLPLib oracle: `=best= 2.5409191380`, `=bestdual= 2.5409129340`. For a minimize
problem the certified dual (lower) bound 3.6142 exceeds a feasible objective 2.5409
— an impossible underestimator, a genuine false-optimal certificate.

## Per-node trace — WHERE the bound crossed 2.5409

The oracle optimum (`minlplib/sol/kall_circles_c8a.p2.sol`, obj 2.540919) maps
cleanly into discopt's flat variable order and lies inside the root box. Tracking
"is the optimal point still in this node's box?" through every per-node reduction:

| stage | result on the optimum-containing box |
|---|---|
| root McCormick LP bound | **valid** (`-1e-9 ≤ 2.5409`) — root relaxation sound |
| Python nonlinear tightening (`nonlinear_bound_tightening.py`, all rules) | never excludes the optimum |
| linear-Jacobian + nonlinear per-node FBBT (`solver.py::_tighten_node_bounds_with_status`) | never excludes the optimum |
| interval bound (`_compute_interval_bound`) | never > 2.5409 on the optimum box |
| Rust FBBT square/sum/sub backprop (`crates/discopt-core/src/presolve/fbbt.rs`) | verified sound (reverse-convex `x²≥L` relaxed to the hull; no outer tightening) |
| **per-node McCormick LP (`MccormickLPRelaxer.solve_at_node`)** | **declares `status="infeasible"` on a node whose box CONTAINS the optimum** ← bug site |

The first node whose *verdict* removes the region containing 2.5409 is the McCormick
LP `solve_at_node` returning `infeasible` on a box that provably contains a feasible
point at obj 2.5409. A McCormick LP is a *valid outer relaxation*, so a truly-empty
relaxed polytope would be a rigorous fathom — but here the LP is **feasible** and
the simplex's `infeasible` is a numerical artifact.

## Root cause classification: **(a) unsound per-node relaxation**

Dumping every `infeasible` verdict of `solve_lp_warm_std` during the buggy solve and
re-solving each with HiGHS: **all 7 were FALSE — HiGHS finds every one feasible.**
Re-solving them through the in-house `solve_lp_warm_std` (Rust dual simplex):

| LP | warm-started? | coef spread | Rust cold verdict | Rust equilibrated verdict | HiGHS |
|---|---|---:|---|---|---|
| lpw_1 | yes | 9.8e3 | infeasible, farkas=**False** | optimal | feasible |
| lpw_2 | no | 9.8e3 | infeasible, farkas=**False** | optimal | feasible |
| lpw_3–7 | no | 4.6e2 | infeasible, farkas=**False** | **infeasible, farkas=False** | feasible |

Findings:
- The in-house dual simplex returns `infeasible` with **`farkas_certified=False`**
  (it cannot produce a valid Farkas ray) on these lifted relaxations — i.e. its own
  infeasibility "proof" does not check out.
- Coefficient spread is only ~1e2–1e4. The existing false-infeasible *equilibration*
  re-verify only triggers above `_RELAX_FALSE_INFEAS_TRIGGER = 1e3` (so lpw_3–7 skip
  it entirely), and even when it *does* fire the same simplex re-confirms the false
  infeasible (lpw_3–7 equilibrated stay infeasible) — **equilibration is not a
  sufficient soundness guard**.
- Every affected node's box contains the true optimum. Fathoming these nodes removes
  the optimum-containing region; the surviving nodes' bounds then exceed 2.5409, and
  the tree certifies 3.6142 as "optimal".

Underlying: the warm-started/ill-conditioned dual simplex can terminate at an
`Infeasible` verdict whose Farkas ray does not certify emptiness
(`crates/discopt-core/src/lp/simplex/dual.rs::run_dual` emits `LpStatus::Infeasible`
without a self-consistent ray and does not cold-fall-back on that). The **Python LP
boundary trusted it** at three points in `mccormick_lp.py`:
- cold path `if res.status == "infeasible": return infeasible` (regardless of
  `res.farkas_certified`);
- incremental `_reverify_incremental_infeasible`: `if not ill: return infeasible`
  (trusts a well-scaled-looking infeasible with no ray) and
  `if status == "infeasible": return infeasible` after equilibration (ignoring the
  `_farkas` flag it computed).

## The fix (general — the class, not the instance)

`python/discopt/_jax/mccormick_lp.py`, three coupled changes; **no name/shape
special-casing**:

1. **Cold path** (`solve_at_node`): fathom on `infeasible` **only if
   `res.farkas_certified`**; otherwise return `status="numerical"` (a non-fathoming
   status), so the driver keeps the node open on its inherited (valid) parent bound
   and branches — exactly as for an `iteration_limit`/`numerical` exit.
2. **`_reverify_incremental_infeasible`**: removed the two uncertified-infeasible
   trusts. It now equilibration-re-solves and accepts `infeasible` **only** on a
   verified Farkas ray; a recovered feasible point returns `optimal`; anything else
   returns `None` (cold fallback). Never trusts an uncertified infeasible.
3. **Incremental fast path** (`_try_incremental_node`): on a warm-started
   uncertified `infeasible`, re-solve the identical assembled system **cold**
   (`in_basis=None`) once — a cold solve is authoritative, so a warm-start artifact
   vanishes: a feasible re-solve returns its valid bound (node stays open), a
   Farkas-certified infeasible fathoms. Then defer to the (now Farkas-gated) reverify.

This forgoes a *possible* prune, never a valid bound — sound by construction. It is
strictly the CLAUDE.md §3 "refuse loudly rather than approximate": without a Farkas
proof of emptiness, the solver does not fathom.
